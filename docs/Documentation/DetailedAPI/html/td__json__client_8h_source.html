9/**
10 * \file
11 * C interface for interaction with TDLib via JSON-serialized objects.
12 * Can be used to easily integrate TDLib with any programming language which supports calling C functions
13 * and is able to work with JSON.
14 *
15 * The JSON serialization of TDLib API objects is straightforward: all API objects are represented as JSON objects with
16 * the same keys as the API object field names. The object type name is stored in the special field "@type" which is
17 * optional in places where type is uniquely determined by the context.
18 * Fields of Bool type are stored as Boolean, fields of int32, int53, and double types are stored as Number, fields of
19 * int64 and string types are stored as String, fields of bytes type are base64 encoded and then stored as String,
20 * fields of array type are stored as Array.
21 *
22 * The main TDLib interface is asynchronous. To match requests with a corresponding response, the field "@extra" can
23 * be added to the request object. The corresponding response will have an "@extra" field with exactly the same value.
24 * Each returned object will have an "@client_id" field, containing the identifier of the client for which
25 * a response or an update was received.
26 *
27 * A TDLib client instance can be created through td_create_client_id.
28 * Requests can be sent using td_send and the received client identifier.
29 * New updates and responses to requests can be received through td_receive from any thread after the first request
30 * has been sent to the client instance. This function must not be called simultaneously from two different threads.
31 * Also, note that all updates and responses to requests must be applied in the order they were received for consistency.
32 * Some TDLib requests can be executed synchronously from any thread using td_execute.
33 * TDLib client instances are destroyed automatically after they are closed.
34 * All TDLib client instances must be closed before application termination to ensure data consistency.
35 *
36 * General pattern of usage:
37 * \code
38 * int client_id = td_create_client_id();
39 * // share the client_id with other threads, which will be able to send requests via td_send
40 *
41 * const double WAIT_TIMEOUT = 10.0; // seconds
42 * while (true) {
43 * const char *result = td_receive(WAIT_TIMEOUT);
44 * if (result) {
45 * // parse the result as a JSON object and process it as an incoming update or the answer to a previously sent request
46 * }
47 * }
48 * \endcode
49 */

109/**
110 * \file
111 * Alternatively, you can use old TDLib JSON interface, which will be removed in TDLib 2.0.0.
112 *
113 * Objects and functions serialization to JSON is the same for both JSON interfaces.
114 *
115 * The main TDLib interface is asynchronous. To match requests with a corresponding response a field "@extra" can
116 * be added to the request object. The corresponding response will have an "@extra" field with exactly the same value.
117 *
118 * A TDLib client instance can be created through td_json_client_create.
119 * Requests then can be sent using td_json_client_send from any thread.
120 * New updates and request responses can be received through td_json_client_receive from any thread. This function
121 * must not be called simultaneously from two different threads. Also, note that all updates and request responses
122 * must be applied in the order they were received to ensure consistency.
123 * Given this information, it's advisable to call this function from a dedicated thread.
124 * Some service TDLib requests can be executed synchronously from any thread by using td_json_client_execute.
125 * The TDLib client instance can be destroyed via td_json_client_destroy.
126 *
127 * General pattern of usage:
128 * \code
129 * void *client = td_json_client_create();
130 * // somehow share the client with other threads, which will be able to send requests via td_json_client_send
131 *
132 * const double WAIT_TIMEOUT = 10.0; // seconds
133 * int is_closed = 0; // should be set to 1, when updateAuthorizationState with authorizationStateClosed is received
134 * while (!is_closed) {
135 * const char *result = td_json_client_receive(client, WAIT_TIMEOUT);
136 * if (result) {
137 * // parse the result as JSON object and process it as an incoming update or an answer to a previously sent request
138 * }
139 * }
140 * td_json_client_destroy(client);
141 * \endcode
142 */
143
